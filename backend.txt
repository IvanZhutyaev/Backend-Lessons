FRONTEND vs BACKEND
Frontend - клиентская часть продукта. В случае с сайтом ее формирует и выводит на экран браузер

Backend - внутреняя часть продукта, которая находится на сервере и скрыта от пользователей


500 Internal Server Error - ошибка, из-за дисконекта между сервером и фронтом

HTTP - протокол передачи гипертекста. Это набор правил, по которым данные в интернете передаются между разными источниками, обычно между компьютерами и серверами. Интетнет-протокол HTTP - это шаблон, по которому формируется запрос на передачу данных, а затем передаются интерент-страницы, видео, аудио и текст

Структура HTTP-сообщения всегда одинаковова:
1)Стартовая строка, в которой определяется адрес, по которому отправляетсмя запрос, и тип сообщения. Указывается метод, который .оперделяет действия при получении этого сообщения. Это может быть чтение данных, их отправка, изменение и удаление.
2)Заголовки, в которых прописаны определённые параметры сообщения
3)Тело запроса, текст сообщения - данные, которые передаются

Основные методы HTTP
1)GET - используется для запроса некоторых данных с сервера(получает, но не изменяет|сохраняется в истории)
2)POST - отправка данных на сервер для создания или обновления запрашиваемого ресурса(не сохраняется в истории|в ответ на него сервер выполнит фоновый процесс\отчитается)
3)PUT - также отправка данных на сервер для создания или обновления(не требует от сервера ответа)
4)PATCH - используется для частичного изменения ресурса(инструкция по обновлению нужной части)
5)DELETE - используется для удаления запрашиваемого ресурса(полностью удаляет ненужную информацию)

Работа HTTP-протокола максимально проста - клиент передаёт запрос, сервер формирует ответ и передаёт обратно

REST - Representational State Transfer - передача состояния представления
REST определяет как компоненты должны взаимодействовать друг с другом(отправка запроса - клиент|обработки - сервер)

Принципы REST:
1)Приведение архитектуры к модели клиент-сервер(нужно разграничивать обязанности клиента и сервера)
2)Отсутствия состояния(запрос должен иметь ВСЮ необходимую информацию)
3)Кэширование(клиенты могут выполнять кэширование ответов сервера)
4)Единообразие интерфейса(унифицированный интерфейс)
5)Слои(применение промежуточных серверов для балансировки нагрузки и распределения кэширования|Прокси-сервер, микро-сервис, балансировщик)

Методы REST API
CRUD - Create, Read, Update, Delete

Endpoint - конечная точка веб-сервиса, предоставляющая доступ к определённому ресурсу или набору ресурсов



SPRING BOOT - ФРЕЙМ ДЛЯ РАЗРАБОТКИ БЕКА НА JAVA
Фреймворк - заготовка, готовая модель в программировании для быстрой разработки, на основе которой можно дописать собственный код. Он задаёт структуру, определяет правила и предоставляет необходимый набор инструментов для создания проекта

Web App - программа с определённым набором функционала, использующая в качестве клиента браузер

Составляющие Sprint Framework:

Spring Core Container - модуль с базовыми функциями фреймворка(внедрение зависимостей и управление жтзненным циклом бинов)
Sprint AOP - модуль, котороый позволяет реализовыватиь аспектно-ориентированное программирпование в приложениях
Spring Data Access/Integration - набор модулей обеспечивает доступ к данным из различных источников, включая реляционные БД
Spring WEB - модуцли, предназначенные для разработки веб-приложений и  веб-сервисов
Spring Test - модуль для тестирования
Spring Security - модуль для обеспечения безопасности в приложениях
Spring Transactions - модуль поддержка транзаций в web-приложениях
